# Aurora - Geopolitical Intelligence Platform

## Project Overview
Aurora monitors active war and conflict prediction markets on Polymarket, enriches them with live trading signals and news context, and uses Google Gemini to generate geopolitical analysis. The output is served via a FastAPI backend and visualised on an interactive Next.js dashboard with a 3D globe.

## Tech Stack
**Backend**
- **Python 3** with **FastAPI** (async, auto-populates data on startup)
- **Google Gemini** (`google-genai`) for geocoding and LLM analysis
- **httpx** for async HTTP requests
- **uvicorn** as the ASGI server, port **9878**

**Frontend**
- **Next.js 15.2** with App Router
- **React 19** + **TypeScript** + **Tailwind CSS 4**
- **globe.gl** for the 3D globe visualisation

## Running Locally

### Backend
```bash
source venv/bin/activate
uvicorn app:app --reload --port 9878
```
On startup, the server automatically runs the full populate pipeline to refresh `markets.json`, then begins serving requests. If populate fails, it falls back to any existing `markets.json` on disk.

### Frontend
```bash
cd aurora---geopolitical-intelligence-main
npm install
npm run dev
```
Frontend runs on `http://localhost:3000`. On load it tries:
1. `http://localhost:9878/markets` — live backend
2. `/api/markets` — local Next.js route reading `markets.json` from disk
3. Hardcoded mock data (last resort fallback)

## API Endpoints (Backend)
- `GET /` — Health check
- `GET /markets` — Returns enriched `markets.json`
- `GET /polymarket/events?limit=10` — Live proxy to Polymarket Gamma API

## Code Conventions
- Use **async** endpoints for all I/O-bound operations
- Raise `HTTPException` for API errors
- Keep `.env` out of version control; use `.env.example` as template
- Frontend data transformations live in `lib/data.ts`; state in `lib/store.tsx`
